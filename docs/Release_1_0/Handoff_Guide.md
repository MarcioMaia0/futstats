## O que entender antes de codar

- O público casual é prioridade de adoção.
- O usuário analítico é prioridade de profundidade e retenção.
- Social não é extra; é porta de entrada.
- Scout avançado não pode bloquear uso simples.
- Legado histórico não é o principal argumento inicial.
- Código, banco e APIs usam inglês.
- UI pode usar Technical, Varzea ou Resenha.
- Dados canônicos não devem depender de texto de interface.

## Como usar a documentação

Para implementar um módulo, leia primeiro:

1. domínio correspondente;
2. especificação de backend;
3. especificação de API;
4. especificação de banco;
5. especificação de UX/frontend;
6. testes de aceite;
7. ADRs relacionados.

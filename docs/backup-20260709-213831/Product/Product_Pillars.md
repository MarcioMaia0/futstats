# Product Pillars

## Objetivo

Documentar os quatro eixos que sustentam o FUTSTATS.

## 1. Organização

Organização é tudo que ajuda o jogo a acontecer.

### Exemplos

- Criar time.
- Cadastrar elenco.
- Registrar partida.
- Controlar agenda.
- Registrar quadra.
- Registrar adversário.
- Registrar arbitragem.
- Controlar presença.

### Valor

Ajuda diretores e organizadores a reduzirem caos operacional sem exigir profissionalização completa.

## 2. Experiência

Experiência é tudo que torna o jogo divertido, compartilhável e culturalmente conectado à várzea.

### Exemplos

- Cards de resultado.
- Perfis de atletas.
- Resenha.
- Reações.
- Comentários.
- Linguagem configurável.
- Temas do time.
- Feed social.

### Valor

É a principal porta de entrada para adoção inicial.

## 3. Inteligência

Inteligência é tudo que transforma dados em aprendizado.

### Exemplos

- Estatísticas.
- Rankings.
- Plus/Minus.
- Scout.
- Relatórios.
- Desempenho por quadra.
- Desempenho por adversário.
- Avaliação de arbitragem.
- Insights de IA.

### Valor

Atende times, técnicos, analistas e estudiosos que desejam melhorar performance.

## 4. Legado

Legado é o histórico construído naturalmente pelo uso.

### Exemplos

- Histórico de partidas.
- Recordes.
- Linha do tempo.
- Artilheiros históricos.
- Conquistas.
- Fotos.
- Vídeos.
- Memória do time.

### Valor

Fica mais forte quanto mais tempo a plataforma é usada.

## Relação entre os pilares

```text
Organização → gera dados
Experiência → gera engajamento
Inteligência → gera aprendizado
Legado → preserva o valor acumulado
```

## Regra fundamental

Nenhum pilar deve obrigar o usuário a usar outro. Um time pode usar apenas Experiência e Organização. Outro pode avançar para Inteligência. Ambos são usuários válidos.

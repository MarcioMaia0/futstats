# Future Ideas

## Objetivo

Guardar ideias de evolucao futura que fazem sentido para o FUTSTATS, mas que ainda nao devem ser tratadas como contrato canonico de produto, API, banco ou frontend.

## Regra de leitura

- Esta pasta nao e fonte canonica.
- Nada aqui deve sobrescrever `Table_Spec_*`, APIs oficiais, ADRs aceitas ou telas ja fechadas.
- Uma ideia documentada aqui pode:
  - ser promovida depois para documento canonico;
  - ser adaptada;
  - ser descartada;
  - ser implementada parcialmente.

## Quando usar esta pasta

- quando a ideia ja ficou boa o suficiente para nao se perder;
- quando ainda nao vale consolidar como regra oficial;
- quando ha valor em preservar a intencao para discussao futura.

## Como promover uma ideia

Quando uma ideia sair do campo hipotetico e virar decisao real:

1. mover o contrato para a fonte canonica adequada;
2. atualizar banco, API, dominio, frontend e ADRs quando necessario;
3. deixar este arquivo apenas como referencia historica ou remover a ideia daqui.

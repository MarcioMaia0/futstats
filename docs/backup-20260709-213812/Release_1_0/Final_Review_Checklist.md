# Final Review Checklist

## Produto

- A proposta de valor casual-first está clara?
- O usuário casual consegue entender por que usaria o app no primeiro jogo?
- O usuário analítico entende o caminho para aprofundamento?
- O social aparece como porta de entrada?
- O legado aparece como consequência, não promessa principal inicial?

## UX

- O fluxo de partida rápida está simples?
- O usuário consegue compartilhar resultado com poucos toques?
- Campos avançados estão escondidos ou opcionais?
- Linguagem e tema são configuráveis?

## Tecnologia

- Tabelas e APIs usam inglês técnico?
- Domínios estão separados?
- Autenticação, usuário e jogador são entidades distintas?
- Regras de permissões estão claras?

## Dados

- Estatísticas simples funcionam com poucos dados?
- Estatísticas avançadas possuem pré-condições?
- Recalcular estatísticas após edição está documentado?

## Segurança

- Papéis por time estão claros?
- Privacidade de atletas e times está considerada?
- Moderação social está prevista?

## Operação

- Existe estratégia de backup?
- Existe observabilidade mínima?
- Existe plano de incidentes?

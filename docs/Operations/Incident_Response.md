# Incident Response


## Objetivo

Definir resposta a incidentes.

## Incidentes possíveis

- indisponibilidade;
- perda de dados;
- falha de login;
- vazamento;
- abuso social;
- falha na geração de cards;
- inconsistência estatística.

## Regras

1. Registrar incidente.
2. Identificar impacto.
3. Comunicar quando necessário.
4. Corrigir causa raiz.
5. Criar ação preventiva.

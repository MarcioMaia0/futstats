# Component Guidelines

## Objetivo

Definir padrões para componentes.

## Tipos

- Layout components.
- Domain components.
- Form components.
- Feedback components.
- Data visualization components.
- Social components.

## Regra

Componentes reutilizáveis devem receber dados canônicos e resolver apresentação via camada de experiência.

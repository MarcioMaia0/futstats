# Feature Prioritization

## Objetivo

Definir critérios para priorizar funcionalidades.

## Perguntas obrigatórias

1. Entrega valor no primeiro uso?
2. Aumenta recorrência?
3. Ajuda o social/resenha?
4. Ajuda organização sem burocratizar?
5. Gera dados úteis para o futuro?
6. Pode ser opcional para usuários casuais?
7. Aumenta profundidade para times maduros?
8. Preserva a identidade da várzea?

## Matriz de prioridade

### Alta prioridade

Funcionalidades simples, compartilháveis e recorrentes.

### Média prioridade

Funcionalidades de organização que melhoram retenção.

### Baixa prioridade inicial

Funcionalidades analíticas profundas que dependem de muita base de dados.

## Regra

Nenhum recurso avançado deve atrasar a entrega do fluxo casual do MVP.
